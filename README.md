# business-leyu-archive

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能包含各类信息展示、工具或文档，旨在提供一个统一的、版本可控的静态资源存放空间。

## 目录结构

```
business-leyu-archive/
├── pages/          # 存放所有独立的 HTML 页面文件
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
├── index.html      # 仓库首页，可作为页面导航入口
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件为一个独立页面，命名清晰，便于识别。
- **assets/**：按类型或页面分目录存放资源，保持整洁。
- **index.html**：可自定义为页面列表或引导页，方便浏览。

## 页面归档说明

- 所有页面均为独立的静态 HTML 文件，不依赖后端服务。
- 页面内容可能包含文字、图片、链接等，均以本地或相对路径引用资源。
- 归档过程中可能对原有内容进行必要的格式调整，以保持页面在仓库中的可读性和稳定性。
- 页面文件命名建议使用英文或拼音，避免特殊字符，确保跨平台兼容。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有资源文件请放入 `assets/` 对应子目录。
- 更新页面：直接修改对应文件，建议在 commit 信息中注明修改内容。
- 删除页面：移除相关文件及引用资源，确保仓库无残留无效链接。
- 本仓库不保证所有页面长期可用，但会尽量保持历史版本可回溯。
